# Project Questio: 수리논술 AI 조교 시스템 매뉴얼

본 프로젝트는 **Questio AI Engine**을 기반으로 하며, 클라우드 저장소(GCS)의 시험 자산과 Google Sheets의 데이터 로그를 결합하여 수리논술 채점 업무의 혁신을 목표로 합니다.

---

## 🚀 업무 작업 순서 (Workflow)

### 1. 시스템 접속 및 권한 인증
*   **인증 코드 입력**: 메인 화면에서 본사 발급 코드(`MATHLOGIC-2026-PREMIUM`)를 입력하여 접속합니다.
*   **인프라 활성화**: 인증 성공 시 `questio-assets-storage` 버킷 및 중앙 데이터베이스 동기화 기능이 활성화됩니다.

### 2. 마스터 데이터베이스 구축 (관리자 전용)
*   **GCS 자산 업로드**: 문제지(PDF/IMG)와 해설지 파일을 `questio-assets-storage` 버킷에 업로드하고 공개 URL을 확보합니다.
*   **CSV DB 생성**: 대학교, 연도, 유형, 문제URL, 해설URL이 포함된 CSV 파일을 작성합니다.
*   **DB 동기화**: `Config` 섹션의 설정(톱니바퀴) 아이콘을 클릭하여 작성한 CSV를 업로드합니다. 이제 모든 지점에서 해당 자산을 즉시 호출할 수 있습니다.

### 3. 시험 설정 및 AI 기준 탐색
*   **대학/연도 입력**: 채점할 대상 대학과 연도를 입력합니다.
*   **문제유형 검색**: `문제유형 검색` 버튼을 눌러 GCS에 등록된 세부 유형(예: 자연계열 오전, 의예과 등)을 선택합니다.
*   **자산 자동 로드**: 유형 선택 시 GCS의 문제지와 해설지가 자동으로 'Linked' 상태가 됩니다.
*   **AI 채점 기준 생성**: Gemini AI가 온라인 검색을 병행하여 해당 시험의 공식 채점 가이드라인을 하단 텍스트 영역에 자동으로 요약 기술합니다.

### 4. 학생 등록 및 답안 업로드
*   **학생 정보 입력**: `Candidate Registry`에서 학생의 이름과 결과 보고서를 수령할 이메일을 입력합니다.
*   **답안 스캔본 업로드**: 학생이 작성한 답안지(이미지 또는 PDF)를 드래그 앤 드롭으로 업로드합니다. 여러 명의 학생을 한 번에 등록하여 일괄 처리가 가능합니다.

### 5. AI 일괄 분석 실행
*   **Batch Analysis**: `Execute Questio-Engine` 버튼을 클릭합니다.
*   **멀티모달 분석**: AI가 (1)공식 해설지 (2)채점 가이드라인 (3)학생 답안 이미지를 동시에 분석하여 수학적 논리 전개의 타당성을 검토합니다.

### 6. 결과 검토 및 보고서 발행
*   **보고서 확인**: 생성된 HTML 보고서에서 총점, 항목별 점수, 강약점 분석을 확인합니다.
*   **다양한 포맷 내보내기**:
    *   **PDF/HTML**: 오프라인 보관용 파일 생성.
    *   **HWP 복사**: 한글(HWP) 문서에 서식 그대로 붙여넣기 가능.
    *   **이메일 전송**: 학생에게 즉시 분석 결과 발송.

### 7. 클라우드 성과 데이터 동기화
*   **Sheets 자동 저장**: 모든 채점 결과는 Google Sheets에 실시간으로 기록됩니다.
*   **Growth Matrix**: `상담용 통계` 버튼을 클릭하여 클라우드에 누적된 데이터를 바탕으로 한 학생의 성적 추이 및 취약 개념 변화를 시각적으로 확인하고 상담에 활용합니다.

---

## 🛠 기술 스택 및 보안
*   **AI Model**: Gemini 3.0 Pro/Flash (Thinking Mode 활성화)
*   **Storage**: Google Cloud Storage (`questio-assets-storage`)
*   **Database**: 클라우드(Google Sheets) + 로컬(Browser IndexedDB) 하이브리드
*   **Security**: 발급 코드 기반의 Enterprise 암호화 접속 및 GCS 접근 제어
